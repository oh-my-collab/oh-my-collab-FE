# 작업 지시 사항

이 문서는 이 저장소에서 작업할 때 반드시 지켜야 하는 기본 규칙을 정의한다.

## 1. 커밋 원칙
- 하나의 작업(작은 단위)당 커밋 1개를 원칙으로 한다.
- 서로 다른 목적의 변경사항을 하나의 커밋에 섞지 않는다.
- 커밋 메시지는 변경 의도가 명확하게 드러나야 한다.

## 2. 브랜치 원칙
- 새로운 기능 추가 또는 큰 단위 작업을 시작할 때는 반드시 새 브랜치를 만든다.
- 브랜치 이름은 작업 목적이 드러나도록 작성한다.
- 작업 완료 후 `main` 기준으로 Pull Request를 생성한다.

## 3. 지시 사항 및 컨벤션 준수
- 앞서 주어진 지시 사항을 우선순위 높게 항상 준수한다.
- 작업 전 프로젝트의 기본 컨벤션(코드 스타일, 파일 구조, 테스트 방식)을 확인한다.
- 컨벤션이 불명확하면 임의로 진행하지 말고 문서화 후 정리한다.

## 4. 검증 원칙
- 완료 주장 전 최소 검증을 수행한다.
- 가능한 경우 `lint`, `test`, `build`를 실행해 결과를 확인한다.
