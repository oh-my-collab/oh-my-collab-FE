# 관리자 설정 가이드

## 0. 진입 및 화면 구조
- 기본 진입 경로: `/overview`
- 핵심 운영 화면:
  - 요약: `/overview`
  - 데드라인: `/deadlines`
  - 작업 실행: `/tasks`
  - 팀 역할: `/team`
  - 관리자 콘솔: `/admin`

## 1. 관리자 권한 지정(Owner 전용)
- 경로: `관리 > 관리자 권한 관리`
- 전제: 작업 수행 계정이 `owner`여야 함
- 절차:
  1. 대상 사용자 행에서 역할을 `admin` 또는 `member`로 선택
  2. `저장` 버튼 클릭
  3. 변경 후 목록 재조회로 반영 확인
- 감사 로그:
  - 이벤트: `membership_role_updated`
  - 저장 위치: `admin_audit_logs`

## 2. 평가 주기 생성 및 가중치 설정
- 경로: `관리 > 평가 주기와 가중치`
- 필수 입력:
  - 주기 이름
  - 시작일 / 종료일
  - 가중치(실행/문서/목표/협업)
- 규칙:
  - 가중치 총합은 100이어야 생성 가능
  - 주기 상태는 `draft/open/closed`로 변경 가능

## 3. 증거팩 해석 가이드
- 경로: `관리 > 구성원 리뷰와 증거팩`
- 증거팩 구성:
  - Raw: 기간 내 실제 활동량
  - Normalized: 같은 주기 내 구성원 상대 정규화 값
  - Score Preview: 가중치 반영 참고 점수
- 주의:
  - `Score Preview`는 자동 추천값
  - 최종 평점은 관리자가 수동 입력 후 확정
  - `확정 및 잠금` 이후 리뷰는 수정 불가

## 4. 내보내기/보존정책 운영
- 내보내기:
  - 경로: `관리 > 구성원 리뷰와 증거팩`
  - 포맷: `CSV`, `PDF`
  - 이벤트: `performance_exported`
- 보존정책:
  - 기본 기간: 12개월
  - 경로: `관리 > 운영 설정`
  - 운영 권장:
    - 분기별 보존 정책 점검
    - 삭제 전 팀 공지 및 백업

## 5. Jira형 운영 화면 사용법
- 작업 워크벤치(`/tasks`)
  - 탭: 백로그 / 스프린트 / 칸반 / 블로커
  - 백로그 정렬 변경은 즉시 저장되며 우선순위 재배치에 활용
- 데드라인(`/deadlines`)
  - 14일 타임라인으로 마감 집중 확인
  - 지연, D-3 이내, 블로커를 위험 항목으로 분리 확인
- 팀 역할(`/team`)
  - 전원 조회 가능
  - 오너만 역할 변경 가능(변경 시 감사 로그 기록)

## 6. 운영 체크리스트
- owner/admin 권한 분리가 실제 조직 운영과 일치하는지 확인
- 평가 확정 전 리뷰 잠금 여부 재확인
- 내보내기 파일 접근권한(사내 저장소/권한) 통제
